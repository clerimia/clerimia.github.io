chat completions API 协议

发表于2026-06-29|更新于2026-07-01|技术AIAPI协议
|浏览量:

核心特点：

  • 无状态，需要手动传递完整的 messages 对话历史
  • 基于角色的对话模型：system/user/assistant/tool
    • 消息顺序约束：system -> user -> assistant -> tool -> user
    • 每轮对话必须以 user 或者 tool 消息结尾（发起新请求时）
    • tool 消息必须紧跟在 tool_calls 和 assistant 消息后
    • 并行工具调用时，所有工具结果必须全部回传后才能继续对话
  • LLM API的事实标准，现在大部分的AI API都是提供了chat completions api兼容的，而且Open AI也称这个为永久兼容的。

一请求结构总览

1. 请求URL规范, baseURL

采用 chat completions api 协议的网关一般的端点格式为：
POST https://modelverse/v1/chat/completions

2. 请求头规范

HTTP Headers 中一般要携带
Authorization: Bearer <api-key> 来进行身份认证和Token审计。
Content-Type: application/json 来接收响应结果

3. 请求体规范

请求体的结构较为复杂，一般是一个json文本的格式组织的，下面分为几类来讲

3.1. 基本必填的

  • model：模型名称
  • messages：消息列表，消息协议
    • system 消息：
      • role：“system”
      • content：内容
      • name?: 区分不同的system消息
    • user 消息：
      • role：“user”
      • content：可以是数组
        • type：“text” | “image_url” | “input_audio” |
        • image_url?:
          • url: URL | DataURI
        • `audio_url?:
          • url：URL | DataURI
          • format
    • assistant 消息
      • role：“assistant”
      • content: string | null（工具调用时）
      • tool_calls?: 数组
        • id
        • type：“function”
        • function
          • name
          • arguments
    • tool 消息
      • role：“role”
      • tool_call_id：和assistant的toolid对应
      • content：string | []

3.2 生成控制参数

  1. 随机性控制
    1. temperature：number：越低就会越倾向于选择高概率的token，也就是概率抉择
    2. top_p：number： 概率采样, 0.1即考虑前10%，截断低概率token
    3. seed：integer：随机种子
    4. 常用的随机程度对比
  2. 长度控制
    1. max_completion_tokens：最大生成Token数
    2. max_tokens：同上
    3. n：每次请求返回的候选回复数量
    4. stop：遇到指定字符串时停止生成
  3. 重复惩罚
    1. presence_penalty：惩罚已经出现的 token，鼓励引入新话题
    2. frequency_penalty：惩罚高频 token
    3. logit_bias：调整指定 token 的概率。
  4. 概率分析
    1. logprobs：是否响应中返回每个 token 的对数概率，越接近 0 越确定
    2. top_logprobs：每个位置返回概率最高的前 N 个 token 及其概率需要先打开 logprobs

3.3 工具列表和Schema

  • tools：工具定义列表
    • name：required
    • description?
    • strict?
    • parameters?
      • zod 来写
    • strict：boolean 模式约束，开启后严格按照 schema 生成
  • tool_choice?：”none“ | “auto” | “required” | TOOL JSON
    • none：禁止调用工具
    • auto：自行判断，默认
    • required：至少调用一个
    • TOOL JSON：强制调用指定工具
  • parallel_tool_calls：boolean 默认 true 开启并行调用工具

3.4 结构化输出

  • response_format：json
    • type：“text” | “json_object” | “json_schema”
      • text：默认，普通文本输出
      • json_object：输出时合法JSON但是不约束结构
      • json_schema：输入schema：严格要求输出符合 schema strict：true 时100%正确

3.5 流式传输

  • stream ： 默认false，
  • stream_options：{“include”：true} 让最后一帧附带 token 用量统计 usage 对象

最后输出的是 [DONE]

3.6 其它参数

  • user：用户的唯一标识
  • store：是否将本次请求存储用于 Dashboard 查看或者 Fune-tuning
  • metadata：自定义键值对，用于业务标注
  • reasoning_effort：思考强度
  • modalities：输出音频模态
  • audio：音频输出类型

响应结构总览

1. 阻塞式响应

{
	"id": "chatcmpl-xyz789",
	"object": "chat.completion",
	"created": 1712345678,
	"model": "gpt-4o-2024-08-06",
	"choices": [
		{
			"index": 0,
			"message": {
				"role": "assistant",
				"content": null,    // 有 tool_calls 时 content 为 null
				"refusal": null,
				"tool_calls": [
					{
						"id": "call_abc111",              // 唯一调用 ID，回传结果时必须对应
						"type": "function",               // 目前固定为 "function"
						"function": {
							"name": "get_weather",          // 调用的函数名
							"arguments": "{\"city\": \"北京\", \"days\": 3}"  // JSON 字符串，需 json.loads() 解析
						}
					},
					{
						"id": "call_abc222",             // 并行调用第二个工具
						"type": "function",
						"function": {
							"name": "get_weather",
							"arguments": "{\"city\": \"上海\", \"days\": 3}"
						}
					}
				]
			},
			"logprobs": null,
			"finish_reason": "tool_calls"  // 标志：需要执行工具并回传结果
		}
	],
	"usage": {
		"prompt_tokens": 85,
		"completion_tokens": 42,
		"total_tokens": 127,
		"prompt_tokens_details": {
			"cached_tokens": 0,
			"audio_tokens": 0
		},
		"completion_tokens_details": {
			"reasoning_tokens": 0,
			"audio_tokens" 0
			"accepted_prediction_tokens" 0
			"rejected_prediction_tokens" 0
		}
	},
	"system_fingerprint": "fp_abc123"
}

1.1 基本元数据字段

  • id：本次API调用的唯一标识符
  • object：标识返回对象的类型，有 chat.completion 和 response
  • created：UNIX时间戳，代表请求处理时的时间
  • model：模型版本
  • system_fingerprint：系统指纹，代表后端配置。

1.2 核心内容字段 choices

  • choices：是一个数组，包含了模型生成的结果，通常只有一个，但是可以指定n
    • index：结果列表的选项索引
    • logprobs：对数概率值
    • finish_reason：停止生成文本的原因
      • stop：自然结束
      • length：长度限制
      • content_filter：内容过滤
      • tool_calls：结束原因是工具调用
      • null：流式输出中间帧
    • message：消息
      • role：assistant
      • content：null | string | object 当发起工具调用时是null
      • refusal：null 模型拒绝回答时是有的
      • tool_calls：数组，可以并行调用
        • id：工具调用的唯一ID，工具结果对应
        • type：固定为function
        • function：具体函数信息
          • name：函数名
          • arguments：参数信息

1.3 资源消耗统计 usage

  • usage：Token消耗情况
    • prompt_tokens：用户输入（包括历史对话）消耗的token数
    • completion_tokens：生成回复时消耗的token数
    • total_tokens：消耗的总token数
    • prompt_tokens_detail：输入token的细分统计
      • cached_tokens：缓存命中token
      • audio_tokens：音频输入消耗token
    • completion_tokens_detail：输出token的细分统计
      • reasoning_tokens：推理消耗的token
      • audio_tokens：音频输出Token
      • accept_prediction_tokens：接受的预测token
      • rejected_prediction_tokens：被拒绝的预测token
