项目经历

RookieDB - 数据库系统核心模块深度实现

GitHub: https://github.com/clerimia/RookieDB.git

技术栈: Java 11, Maven, JUnit 4, Git | 代码实现: 3,000+ 核心代码行 | 测试: 100+ 单元测试

项目描述: 基于CS186数据库项目框架实现关系型数据库核心引擎，涵盖存储、索引、查询、并发控制、恢复六大子系统，支持完整的ACID事务特性，通过CS186课程全部隐藏测试。

核心负责模块:

查询优化和执行引擎

  • 连接算子实现: 完成块嵌套循环连接(BNLJ)、Grace哈希连接(GHJ)、排序合并连接(SMJ)等多种连接算法
  • 外部算法实现: 实现外部排序和外部哈希算法，支持超出内存容量的大数据集处理
  • 迭代器模型: 采用火山模型实现惰性求值，支持流水线式查询执行
  • 基于成本的动态规划优化: 实现三阶段查询优化器，通过枚举左深连接树选择最优执行路径
  • 谓词下推优化: 将选择条件下推到扫描操作符，支持索引扫描选择

存储引擎与索引

  • B+ 树索引: 实现完整的单列B+树索引，支持插入、删除、点查询、范围查询和批量加载
  • 批量加载优化: 通过预排序和自底向上构建，减少节点分裂次数
  • 页目录管理: 实现堆文件结构，使用页目录管理记录分配，支持定长记录存储
  • 缓冲池管理: 实现LRU和Clock页面置换算法，支持写回缓存策略

事务管理

  • 并发控制: 实现严格两阶段锁协议（Strict 2PL），支持多粒度锁机制，实现了可串行化隔离级别
  • 锁机制: 支持5种锁类型（S/X/IS/IX/SIX），实现锁兼容性矩阵和意向锁规则
  • 恢复机制: 完整实现 ARIES 三阶段恢复算法（分析阶段→重做阶段→撤销阶段）
  • 持久性保证: 通过WAL（Write-Ahead Logging）和日志刷盘机制，实现了事务的原子性和持久性

茵蒂克丝(Index)-校园智能知识库管理系统

GitHub: https://github.com/clerimia/Index.git

技术栈: Spring Boot, Java 17, Spring Security, MyBatis, MySQL, Redis, ElasticSearch, Kafka, Apache Tika, WebFlux

项目描述: 基于 RAG（检索增强生成）技术的校园知识库管理系统，支持多院系/班级/社团的知识共享、隔离与智能问答。采用微服务架构，实现文档智能解析、语义向量检索、AI 问答等功能。

核心指标:

  • 日均处理文档量：500+
  • 问答响应时间：< 2s（首字延迟1-2s）
  • 用户并发：支持100+同时在线用户
  • 知识库容量：支持百万级文档存储

主要贡献:

  • 负责 RAG 智能问答核心模块，设计混合检索方案（向量语义 + BM25 关键词）
  • 实现大文件分片上传与断点续传，使用 Redis Bitmap 记录分片上传状态
  • 设计 WebSocket 流式对话，配合 WebFlux 接收 LLM SSE 响应，首字延迟从 15s 降至 1-2s
  • 实现多租户权限隔离，基于组织标签的 Elasticsearch Filter Query，支持层级继承的权限控制
  • 系统架构设计：采用分层架构（接入层、业务层、数据层）
